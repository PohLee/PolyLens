# PolyLens Artifact Storage

Use this subtree for markdown artifacts generated by PolyLens.

For the full product documentation, see [../../README.md](../../README.md).
For the docs index, see [../README.md](../README.md).

## Path Rules

- Store saved executive review briefs in `docs/polylens/reviews/`
- Store saved pre-fight reports in `docs/polylens/pre-fight/`
- Store plans and proposals in `docs/polylens/plans/`
- Store generic markdown memos, working notes, and memory-style outputs in `docs/polylens/memory/`
- Store other supporting notes in `docs/polylens/notes/`

Recommended project-memory references inside `docs/polylens/memory/`:

- `project-context`: durable facts about the company, product, mission, vision, constraints, and non-negotiables
- `project-preferences`: default PolyLens behavior for that project such as lens defaults, checklist defaults, and template preferences
- `decision-history`: durable prior decisions, rationale, and revisit triggers

## Filename Rule

- Use `YYMMDD_slug_rN.md`
- Example: `260405_selection-logic_r1.md`
- `YYMMDD` is the creation date
- `slug` is a clear, descriptive lowercase topic name, not an overly short label
- Prefer 3-8 meaningful words joined with hyphens when practical
- Make the slug readable on its own, for example `pricing-tier-restructure` instead of `pricing` and `selection-logic-routing-fix` instead of `fix`
- `rN` is the revision number for that dated slug

Examples:

- `docs/polylens/memory/260405_selection-logic_r1.md`
- `docs/polylens/reviews/260405_mongodb-migration_r2.md`
- `docs/polylens/pre-fight/260405_pricing-strategy_r1.md`

Use the subdirectories in this tree for saved artifacts only; keep general product documentation in [../../README.md](../../README.md).
